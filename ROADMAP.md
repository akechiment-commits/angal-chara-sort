## eraあんガルとは

**あんさんぶるガールズ！！**（君咲学院が舞台の学園カードゲーム、2018年サービス終了）を題材にしたeraゲーム。eratohoJ（東方Jシステム）をベースに以下の要素を実装する：

| 要素 | 内容 |
|---|---|
| ベースシステム | eratohoJのコミュ・パラメータ・時間管理システム |
| 舞台 | 君咲学院（3学年×3クラス、9クラス編成） |
| キャラ数 | 71人 |
| 顔グラ | wikiから取得した画像をemuera-anchor形式で表示 |
| その他画像 | AIで生成 |
| シナリオ | ゲームアプリから抽出したテキストをera形式に変換 |
| 学校生活要素 | 時間割・授業・部活・委員会・学校行事 |
| イベント | ソシャゲ原作の18本のイベントをシナリオとして実装 |

## これからやること一覧

### フェーズ1：素材集め（主にユーザーがやること）

#### シナリオ抽出（ユーザー作業）
1. Android端末に `jp.co.happyelements.kimisakimemories` をPlayストアからインストール
2. **APK Extractor**（無料アプリ）をインストールして起動
3. あんガルのAPKファイルを取り出してPCに転送
4. PCで **AssetStudio** または **AssetRipper** を使って開く
5. TextAsset型のファイルを探してエクスポート
6. 抽出されたテキストの内容・フォーマットを共有 → 変換スクリプトを作成

#### 顔グラ収集（ユーザー作業）
- `https://hatotank.net/kmskwiki/` のキャラページから各キャラの顔グラ画像を保存
- ファイル名をキャラIDや名前に統一しておくと後の作業が楽になる
- emuera-anchorで表示できる形式（PNG推奨）

#### その他画像（後回しでOK）
- 背景・UIパーツ等をAIで生成
- 作業タイミングは実装が進んでから

### フェーズ2：era-angalの開発（era-angal用セッションで実施）

#### 2-1. eratohoJベースの解析・整理
- 既存のERB/CSVの構造を把握
- 東方キャラ固有の部分（名前・画像パス・パラメータ値等）を特定
- あんガル用に変更が必要な箇所をリストアップ

#### 2-2. キャラクターCSV作成（71人分）
- 各キャラのCSVファイルを作成
- 記載内容：名前・学年・クラス・部活・誕生日・血液型・初期パラメータ等
- クラス編成：
  - 1-A 優等生 / 1-B 個性派 / 1-C 動物園
  - 2-A 明るい / 2-B 問題児 / 2-C 天才
  - 3-A 優秀 / 3-B 活発 / 3-C 個性的

#### 2-3. パラメータ定義（CSV）
- `BASE.CSV`：体力・精神力など基礎ステータス
- `ABL.CSV`：能力値（勉強・運動・音楽・芸術など学園テーマに合わせる）
- `TALENT.CSV`：才能・個性（部活適性など）
- `PALAM.CSV`：対主人公パラメータ（好感度・信頼・親密度等）
- `EXP.CSV`：経験値系

#### 2-4. 顔グラ表示対応
- emuera-anchor用の画像表示コードをERBに組み込む
- `GCREATE` / `GLOADG` / `GDRAWG` コマンドで実装
- キャラCSVに画像ファイルパスを紐付け

#### 2-5. 学校生活システム実装
- 時間帯システム（朝・午前授業・昼休み・午後授業・放課後・夜）
- 曜日・日付管理
- 授業（科目ごとに勉強パラメータ上昇）
- 部活動（各部活の活動・練習シーン）
- 委員会活動
- 学校行事イベント（文化祭・体育祭・修学旅行等）
